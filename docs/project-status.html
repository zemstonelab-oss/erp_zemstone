ZEMSTONE 판촉물 관리 시스템 현황

2026-02-12 기준

1. 지금 되는 것

관리자 (젬스톤)

  • 전체 현황 한눈에 보기 (발주량, 출고량, 잔량, 출고율)
  • 본사 발주 차수 등록/수정/삭제 — 1차~N차 좌우로 넘기면서 조회
  • 사업소에서 온 출고 요청 승인/거절 — 승인하면 자동으로 재고 차감
  • 직접 출고 처리 — 사업소 선택해서 수량 입력 후 출고
  • 배송 관리 — 상태 변경(접수→준비→배송중→완료), 예정일시, 배송 담당자 입력
  • 정산 — 기간/사업소별 출고 금액 조회 + 엑셀 다운로드 (물산에는 안 보임)
  • 사업소/품목/사용자 추가·수정·삭제
  • 활동 기록 — 누가 언제 뭘 했는지 전부 기록
  • 월별 출고 추이 그래프 + 사업소별 비교 차트
  • 잔량 부족 시 자동 알림
  • 엑셀 다운로드 (재고/출고/발주)

삼성물산 (본사)

  • 전체 현황 조회 (보기만 가능, 수정 불가)
  • 차수별 발주 확인
  • 사업소별 소진율 확인
  • 출고 이력 조회
  • 엑셀 다운로드

사업소

  • 내 재고 현황 확인 (품목별 발주량/출고량/잔량)
  • 출고 요청 — 여러 품목 한번에 선택, 희망 배송일/시간 지정, 특이사항 메모
  • 내 출고 이력 + 배송 상태 확인 (지금 어디까지 왔는지)
  • 배송 담당자 연락처 확인
  • 알림 (배송 출발, 배송 완료 등)

2. 아직 안 된 것

기능중요도설명
핸드폰에서 보기납품 전 필수지금은 PC에서만 깔끔하게 보임. 사업소 담당자가 폰으로 볼 수 있게 해야 함
비밀번호 변경납품 전 필수지금 전부 비밀번호 1234. 첫 로그인 때 바꾸게 해야 함
외부 접속 가능하게 배포납품 전 필수지금은 이 컴퓨터에서만 됨. 인터넷으로 접속할 수 있게 서버에 올려야 함
정산서 PDF있으면 좋음엑셀만 있고 PDF 없음. 물산에 제출할 때 깔끔한 정산서
품목 사진있으면 좋음사업소 직원이 품목 구분하기 쉽게. 사진은 나중에 넣으면 됨
반품 처리있으면 좋음불량이나 잘못 보낸 거 반품하면 재고 다시 복구
공지사항나중에전체 사업소에 공지 보내기
문자/카톡 알림나중에배송 출발하면 자동으로 문자. 별도 비용 발생

3. 사장님이 정해주셔야 할 것

1. 업무 플로우가 맞는지 물산 컨펌
물산이 분기별 발주 → 젬스톤이 공장 생산+창고 보관 → 사업소가 필요할 때마다 출고 요청 → 젬스톤이 배송
→ 이 플로우 문서(business-flow.html) 물산 담당자한테 보여주고 맞는지 확인
2. "직접 출고 처리" 페이지 유지할 건지
지금 관리자가 직접 수량 넣어서 출고하는 페이지가 따로 있음. 사업소가 시스템으로 요청하면 필요 없을 수도 있음.
→ A. 유지 (전화/카톡 주문 처리용으로 남김)
→ B. 삭제 (출고 요청 승인으로 통합)
→ C. 나중에 결정
3. 물산도 출고 요청 승인할 수 있게 할 건지
지금은 젬스톤만 승인 가능. 물산은 보기만 됨.
→ A. 이대로 (젬스톤만 승인)
→ B. 물산도 승인 가능하게
4. 접속 주소 (도메인)
외부에서 접속하려면 인터넷 주소가 필요함.
→ A. 회사 도메인 사용 (예: scm.zemstone.co.kr)
→ B. 새 도메인 구매 (연 1~2만원)
→ C. 무료 주소 사용 (예: zemstone-scm.up.railway.app)
5. 사업소 계정은 누가 만들어주는 건지
지금은 관리자가 어드민 페이지에서 직접 만들어줌.
→ A. 이대로 (젬스톤이 계정 만들어서 알려줌)
→ B. 초대 링크 보내서 사업소가 직접 가입
6. 납품 전에 추가로 넣을 기능
위 "아직 안 된 것" 목록에서 선택
추천: 핸드폰 대응 + 비밀번호 변경 + 배포 (이 3개는 필수)
7. 기존 전화/카톡 주문 방식은 어떻게 할 건지
사업소가 지금까지 전화나 카톡으로 주문하던 걸 이 시스템으로 바꿀 건지.
→ A. 완전 대체 (시스템으로만 주문)
→ B. 병행 (시스템 + 전화/카톡 둘 다 가능)
