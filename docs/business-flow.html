ZEMSTONE SCM 비즈니스 플로우

전체 구조

1
삼성물산(본사) — 분기별 대량 발주
연 4회, 분기별로 전체 사업소 물량을 한번에 발주
↓
2
젬스톤 — 생산 요청 + 창고 보관
발주량을 공장에 생산 요청 → 완제품을 젬스톤 창고에 보관
↓
3
사업소 — 수시 출고 요청
사업소 창고가 작아서, 필요할 때마다 소량씩 젬스톤에 출고 요청
↓
4
젬스톤 — 출고 처리
요청 확인 후 창고에서 해당 사업소로 출고. 잔량 자동 차감.

역할별 기능

젬스톤 (ADMIN)

  • 전체 현황 대시보드
  • 발주 차수 등록/관리
  • 출고 처리
  • 출고 요청 확인/처리
  • 사업소/품목 관리
  • 사용자 관리
  • 잔량 알림 기준 설정
  • 엑셀 리포트

삼성물산 (HQ)

  • 전체 현황 조회
  • 차수별 발주 확인
  • 사업소별 소진율
  • 히스토리 조회
  • 엑셀 리포트

사업소 (BRANCH)

  • 내 재고 현황
  • 출고 요청
  • 내 출고 이력
  • 엑셀 다운로드

재고 계산 로직

잔량 = 전체 발주량(모든 차수 합산) - 전체 출고량


예시: 4차까지 미용티슈 서초 발주 총 800박스

→ 출고 총 550박스 → 잔량 250박스 (젬스톤 창고에 보관 중)


차수가 추가되면 잔량에 합산 (리셋 아님)

컨펌 필요 사항

위 플로우가 실제 업무와 맞는지 삼성물산 측 확인 필요

특히: 사업소 출고 요청 승인 권한 (젬스톤만? 삼성물산도?)

추가 발주(분기 외) 프로세스 확인 필요

참고: 현재 시스템에서 "추가발주 요청"은 "출고 요청"으로 변경 예정. 사업소가 하는 건 발주가 아니라, 이미 발주된 물량 중 소량 출고를 요청하는 것.
